## 📋 Project Overview

**GitHub FM** is a modern, feature-rich web-based file manager and code editor for GitHub repositories. It allows users to browse, create, edit, delete, and manage files and folders directly from the browser using the GitHub API.

### Why GitHub FM?
- No need to clone repositories locally
- Full file management from any device
- Beautiful, minimal interface
- Works offline with PWA support
- Syntax highlighting for 12+ languages

### Key Philosophy
- **Simple & Intuitive** – Even non-coders can use it
- **Lightweight** – Single-page application, no backend required
- **Progressive** – Works as a PWA on any device

## 📥 Installation Guide

### Prerequisites
- A modern web browser (Chrome, Firefox, Edge, Safari)
- A [GitHub Personal Access Token](https://github.com/settings/tokens) with `repo` scope
- Internet connection (for initial load and API calls)

### Quick Start (Recommended)
1. **Download all files** from the repository
2. **Extract** to a folder named `github-fm`
3. **Open** `index.html` in your browser
4. **Generate a token** from [GitHub Settings](https://github.com/settings/tokens)
5. **Paste token** and click **Connect**
6. **Select repository** and start browsing!

### PWA Installation (Optional)
1. Open `index.html` in Chrome/Edge
2. Click the **install icon** in the address bar (or menu → Install)
3. The app will open as a standalone window
4. It will work **offline** after first load

### Self-Hosting
```bash
# Clone the repository
git clone https://github.com/your-username/github-fm.git
cd github-fm

# Serve with any static server
npx serve .
# or
python3 -m http.server 8000

# Open http://localhost:8000
```

## 📖 User Guide

### 🔌 Connecting to GitHub
1. Enter your **Personal Access Token** in the top bar
2. Click **Connect** button
3. Wait for "Authenticated as [username]" toast
4. All your repositories will appear in the dropdown

### 📁 Browsing Files
- **Left sidebar** shows the file tree
- Click **folder names** to expand/collapse
- Click **file names** to open in the editor
- Use **🔄 Refresh** button to reload tree
- Use **📉 Collapse All** to collapse folders

### 📝 Editing Files
- Open any text file to edit
- Code will be **syntax highlighted** by language
- Changes are tracked with a **yellow dot** in the tab
- Press **Ctrl+S** or click **💾 Save** to commit changes
- Enter a **commit message** in the modal

### 🗑️ Deleting Files/Folders
- **Files:** Click **🗑 Delete** button in the top bar
- **Folders:** Hover over folder name and click **🗑 icon**
- Confirmation modal will appear
- Deletion is **permanent** (via GitHub API)

### 📜 Viewing History
- Open a file first
- Click **📜 History** button
- See list of commits for that file
- Click any commit to **view old version**

### 📤 Drag & Drop Upload
- Connect to a repository
- **Drag files** from your computer
- Drop them in the **"Drop files here"** zone in the sidebar
- Files upload to the **root** of the repository

### 📝 Markdown Preview
- Open a `.md` file
- Click **👁 Preview** button
- See rendered Markdown
- Click **✏ Edit** to return to editor

### 🌓 Theme Toggle
- Click **🌓 Theme** button to switch between **Dark** and **Light** modes
- Preference is **saved** in your browser

## 📱 PWA Information

### Progressive Web App Features
- **Installable** – Add to home screen on mobile & desktop
- **Offline Support** – Caches all essential files
- **Auto Update** – Detects new version, prompts user to update
- **Cache Management** – Old caches are automatically cleaned
- **Standalone Mode** – Opens without browser chrome

### Service Worker (sw.js)
- **Install Event:** Caches all HTML, CSS, JS, and CDN resources
- **Activate Event:** Deletes old caches when version changes
- **Fetch Event:** Serves from cache first, then network
- **Message Event:** Handles "SKIP_WAITING" command from app

### Cache Strategy
- **Cache First** for static assets (HTML, CSS, JS, CDN)
- **Network First** for GitHub API calls
- **Cache Name:** `github-fm-v{version}` – Changing version triggers clean

### Updating the App
1. Modify any file and **change the CACHE_NAME** in `sw.js` (e.g., `v1` → `v2`)
2. Users will see a **"New version available!"** toast
3. Clicking **"Update"** reloads the page with the new version
4. Old cache is automatically deleted

## 🔧 Development Guide

### Setting Up Local Development
```bash
# Clone the repo
git clone https://github.com/your-username/github-fm.git
cd github-fm

# Start a local server
npx serve .

# Open http://localhost:3000
```

### Code Organization
- **index.html** – Minimal HTML structure, all elements with IDs
- **style.css** – CSS custom properties for theming, component styles
- **script.js** – Modular functions organized by feature:
  - API helpers (api, getUserRepos, getContents, etc.)
  - Connection management (connect, disconnect)
  - File tree rendering (renderTree, toggleDir, etc.)
  - Tab management (openFile, closeTab, renderTabs)
  - Editor operations (initCodeMirror, saveFile, etc.)
  - PWA logic (service worker registration)

### Adding a New Feature
1. Add HTML elements to `index.html` if needed
2. Add styles to `style.css`
3. Add JavaScript functions to `script.js`
4. Bind events in the event binding section
5. Update `sw.js` CACHE_NAME for new release
6. Update this master document

### Debugging Tips
- Open **Developer Tools** (F12)
- Check **Console** tab for logs
- Check **Application → Service Workers** for PWA status
- Check **Network** tab for API errors
